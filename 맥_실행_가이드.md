# Freewill Seedance 2.0 — Mac 실행 가이드

Mac에서는 Windows EXE 대신 **로컬 웹앱**으로 실행합니다.
영상 생성 · R2 업로드 · 캐시 전부 정상 작동하고, 다운로드는 **브라우저 기본 다운로드 폴더**로 저장됩니다.
(트레이 / 자동 업데이트 / 폴더 선택 / 캐시 정리 버튼만 없음 — 핵심 기능은 동일)

---

## 준비 (최초 1회만)

### 1. Node.js 설치
https://nodejs.org → **LTS** 버전 다운로드 → 설치
확인: 터미널 열고 `node -v` → `v20` 이상이면 OK

### 2. 소스 받기
터미널에서:
```bash
git clone https://github.com/productionkhu-tech/freewill-seedance.git
cd freewill-seedance
```
> 이미 받아놨으면 폴더로 들어가 `git pull` 로 최신화.

### 3. `.env` 파일 만들기  ← 환경변수, 이게 전부입니다
프로젝트 폴더 안에 **`.env`** 라는 텍스트 파일을 만들고 아래 5줄을 채웁니다:

```
SEEDANCE_API_KEY=팀_API_키
R2_ENDPOINT=https://xxxxx.r2.cloudflarestorage.com
R2_ACCESS_KEY_ID=R2_액세스_키
R2_SECRET_ACCESS_KEY=R2_시크릿_키
R2_BUCKET=R2_버킷명
```

- **셸 명령(export) 필요 없음.** 그냥 텍스트 파일이면 서버가 자동으로 읽습니다 (`dotenv`).
- 값은 기존 Windows PC에 설정된 환경변수와 **동일**합니다 → 아래 "값 가져오기" 참고.
- `.env`는 git에 안 올라갑니다(`.gitignore` 처리됨). 안전.

**터미널로 만드는 법:**
```bash
nano .env
```
→ 위 5줄 붙여넣고 값 채우기 → `Control+O`, 엔터(저장) → `Control+X`(나가기)

## 실행 (매번)

- **방법 A (쉬움):** Finder에서 **`start.command` 더블클릭**
- **방법 B:** 터미널에서 폴더 들어가 `npm run dev`

→ 잠시 후 기본 브라우저에 **http://localhost:3000** 자동으로 열림.
→ 종료: 터미널 창에서 **Control+C**.

> `start.command`가 "확인되지 않은 개발자"로 안 열리면, 한 번만 터미널에서
> `chmod +x start.command` 실행 후 다시 더블클릭. (git clone 받았으면 보통 바로 됨)

## `.env` 값 가져오기 (기존 Windows PC에서)

Windows PC의 **PowerShell**에서 아래 한 줄을 실행하면, `.env`에 그대로 붙여넣을 형식으로 5줄이 출력됩니다:

```powershell
foreach ($n in 'SEEDANCE_API_KEY','R2_ENDPOINT','R2_ACCESS_KEY_ID','R2_SECRET_ACCESS_KEY','R2_BUCKET') { "$n=$([Environment]::GetEnvironmentVariable($n,'User'))" }
```

→ 출력된 5줄을 복사해서 Mac의 `.env`에 붙여넣으면 끝.

## Windows EXE와 다른 점

| 기능 | Windows EXE | Mac 웹앱 |
|------|-------------|----------|
| 영상 생성 / R2 / 캐시 | O | O |
| 자동 다운로드 토글 | O | O |
| 다운로드 위치 | 지정 폴더 | 브라우저 기본 다운로드 폴더 |
| 트레이 / 폴더 선택 / 캐시 정리 버튼 | O | 없음 (또는 "데스크톱 전용" 표시) |
| 업데이트 | 자동 | `git pull` 후 재실행 |

## 문제 해결

- **3000 포트 충돌**: 다른 게 3000을 쓰면 `.env`에 `PORT=3100` 한 줄 추가 → http://localhost:3100 로 접속
- **키 오류로 서버가 바로 종료**: `.env`의 5개 값 확인 (특히 R2 4개가 다 있는지)
- **의존성 오류**: `rm -rf node_modules && npm install`
- **`bad interpreter` 오류**: 줄바꿈 문제 → 터미널에서 `npm run dev` 로 실행하면 우회됨
